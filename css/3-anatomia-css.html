3. Anatomía CSS

CSS tiene sus propias reglas/normas de escritura que debemos entender y aprender para poder escribir código correctamente.

Reglas CSS

La estructura básica de un código CSS es la siguiente:

selector { propiedad : valor ; }

Donde:

  • Selector: Es el elemento HTML al que vamos a aplicar el estilo.
  • Propiedad: Es la característica visual que queremos modificar (color, tamaño, tipografía...).
  • Valor: Es el valor que le vamos a dar a esa propiedad (rojo, 20px, Arial...).

Toda esta estructura completa se denomina regla predeterminada, comúnmente abreviada como regla y cada parte de esta estructura tiene una funcionalidad específica:

Selector:

Es el elemento al que vamos a aplicar el estilo. Este selector puede ser un elemento HTML directamente (p, a, header ...) o podemos llamarlo mediante su id, arbol genealógico, etc. Más adelante veremos todos los tipos de selectores que existen.

También podemos tener un selector compuesto de varios elementos que compartan las mismas declaraciones:

selector1 , selector2 { propiedad : valor ; }

Declaración:

Se compone de 2 partes: propiedad y valor de la propiedad.

Un selector puede tener varias declaraciones a la vez (por ejemplo, podemos indicar el color, el tamaño y la tipografía de un selector):

selector {
propiedad1 : valor ;
propiedad2 : valor ;
propiedad3 : valor ;
...
}

Propiedad:

Cada una de las opciones de estilo que existen. El color, el tamaño, la tipografía, el borde, etc. todos son propiedades y se indicarán mediante un nombre específico.

Valor de la propiedad:

Indica el valor de la propiedad. Cada propiedad tiene diferentes posibilidades de valores (por ejemplo, para los tamaños se utilizarán unidades de medida y para los colores, diferentes formas de código de color).

Otras particularidades:

A la hora de escribir una Regla CSS también hay que tener en cuenta:

  • Llaves: Las declaraciones de un selector deben estar encapsuladas entre llaves { }
  • Dos puntos: En cada declaración se deben utilizar los dos puntos : para separar la clave (propiedad) de su valor.
  • Punto y Coma: En cada regla se debe usar el punto y coma ; para separar una declaración de la siguiente.

En el siguiente ejemplo vemos una regla sobre el selector p con 5 declaraciones:

p {
color: red;
background: #CCC;
font-weight: bold;
border: 1px solid black;
border-radius: 30px;
}

Y ahora una regla con un selector compuesto de varios elementos que comparten las mismas declaraciones:

p, li {
color: red;
font-weight: bold;
}

Comentarios

Para comentar un código CSS tenemos la siguiente sintaxis:

/* comentarios */

Los comentarios abarcan el bloque completo desde el inicio al final del comentario y no podemos meter un comentario dentro de otro.
