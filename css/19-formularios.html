19. Formularios

Los campos de formulario tienen algunas propiedades, pseudoclases y pseudoelementos específicamente para ellos, y combinadas con otras propiedades generales, nos permiten personalizarlos.

Selectores de atributo

Como se ha explicado anteriormente en el capítulo de Selectores, mediante los selectores de atributo es posible llamar directamente a los diferentes campos de formulario generados por el elemento input: 

input[type="text"]formulario texto
input[type="email"]formulario email
input[type="submit"]formulario submit
...

Pseudo-clases

Para dar estilo a nuestros formularios serán muy útiles todas las pseudo-clases de interacción y formulario.

Pseudo-clase Descripción
:active Elemento activado (por ejemplo, al hacer clic).
:hover Elemento al pasar el cursor por encima.
:focus Elemento enfocado (por ejemplo, al hacer clic o tabular).
:focus-visible Elemento enfocado y visible para el usuario.
:focus-within Elemento que contiene un elemento enfocado.
:enabled Elemento habilitado.
:disabled Elemento deshabilitado.
:read-only Elemento de solo lectura.
:checked Elemento marcado (checkbox/radio).
:indeterminate Elemento en estado indeterminado.
:required Elemento obligatorio.
:optional Elemento opcional.
:valid Elemento con valor válido.
:invalid Elemento con valor no válido.
:in-range Valor dentro del rango permitido.
:out-of-range Valor fuera del rango permitido.

Generalmente se combinan entre sí para poder optimizar el uso del formulario:

  • input:focus:valid
  • input:focus:invalid
  • input:required:valid
  • input:required:focus:valid
  • ...

Pseudo-elementos

Entre todos los pseudo-elementos, tenemos dos específicos para los campos de formulario:

  • ::placeholder
  • ::file-selection-button

Propiedades específicas

Color del cursor

Mediante la propiedad caret-color podemos cambiar el color del cursor mientras estamos dentro de un campo escribiendo el texto.

Habitualmente el cursor es una línea vertical que parpadea, si no se ha modificado el cursor de sistema. El parpadeo no se puede alterar al ser una característica propia del Sistema Operativo. 

input[type="text"]{caret-color:#F00;}

Color destacado

Si queremos personalizarlos al máximo, entonces tenemos que construir desde cero el elemento, con un cargamento de CSS y probablemente JS, cuidando mucho que sea funcional bajo cualquier dispositivo y respetando las normas de accesibilidad. Esto lleva bastante tiempo, incluso si utilizamos alguna librería JS para ello.

Lista de selección

En un elemento select vamos a poder modificar los estilos generales (colores, bordes...) y también el símbolo del desplegable.

Para modificar este símbolo, primero tenemos que eliminar el símbolo estándar con la propiedad appearance y después ubicar nuestro símbolo personalizado (combinando varias propiedades CSS). 

select{
appearance: none;
}

Esta propiedad appearance elimina la apariencia característica de cualquier campo de formulario, por lo que no se debe utilizar a la ligera, ya que también elimina el comportamiento nativo del campo a nivel de navegador y Sistema Operativo, lo que puede provocar errores de funcionamiento y accesibilidad.

Están en proceso una serie de propiedades para poder modificar completamente el aspecto del elemento select de manera más sencilla y sin interferir en la accesibilidad, pero todavía no hay soporte completo.

Si con esto no es suficiente, entonces se tendría que valorar la opción de utilizar una librería JS para modificar por completo su interfaz.
